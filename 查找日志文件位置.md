# 查找日志文件位置

## 问题
日志文件 `/home/admin/gemini-audio-service.log` 不存在。

## 可能的原因
1. 日志文件路径配置错误
2. 日志文件在项目目录下
3. 日志文件在其他位置

## 需要执行的命令

### 1. 查找日志文件
```bash
find /home/admin -name "*.log" -type f 2>/dev/null
find /tmp -name "*gemini*" -type f 2>/dev/null
ls -la ~/gemini-audio-service/*.log 2>/dev/null
```

### 2. 检查进程的日志输出
```bash
ps aux | grep "python3 main.py" | grep -v grep
# 查看进程的 stdout/stderr 重定向
```

### 3. 检查服务启动时的日志
```bash
cat /home/admin/service-startup.log 2>/dev/null
```

### 4. 直接查看 Python 进程的输出
由于服务是用 nohup 启动的，日志可能在：
- 项目目录下
- /tmp 目录下
- 或者没有写入日志文件（只输出到 stdout）
