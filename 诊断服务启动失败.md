# 诊断服务启动失败

服务启动失败（Exit 1），需要查看错误日志。

## 在服务器上执行以下命令：

### 1. 查看最新的错误日志
```bash
tail -50 ~/gemini-audio-service.log
```

### 2. 查看完整的错误信息
```bash
cat ~/gemini-audio-service.log | tail -100
```

### 3. 尝试直接运行查看错误
```bash
cd ~/gemini-audio-service
python3 main.py
```

这会显示实时错误信息，帮助我们诊断问题。

### 4. 检查 Python 和依赖
```bash
python3 --version
cd ~/gemini-audio-service
source venv/bin/activate  # 如果使用虚拟环境
pip3 list | grep -E "fastapi|pydantic|google"
```

### 可能的问题：
1. 缺少依赖包
2. 代码语法错误
3. 环境变量缺失（GEMINI_API_KEY）
4. 端口被占用
