# 如何设置 Deployment Target 为 iOS 16.0

## 📝 详细步骤（适合小白）

### 第一步：打开项目设置

1. **在 Xcode 中，找到项目导航器（左侧面板）**
   - 如果看不到左侧面板，点击顶部菜单栏的 `View` → `Navigators` → `Show Project Navigator`
   - 或者按快捷键 `Cmd + 1`

2. **点击最顶部的项目名称（蓝色图标）**
   - 这个图标通常在最顶部，显示为蓝色文件夹图标
   - 名称应该是 `WorkSurvivalGuide`（或你创建项目时输入的名称）
   - **重要**：要点击项目名称，不是文件夹名称

   ```
   项目导航器示例：
   📁 WorkSurvivalGuide (蓝色图标) ← 点击这个！
      📁 WorkSurvivalGuide
      📁 Models.swift
      ...
   ```

### 第二步：选择 Target

1. **在中间的主编辑区域，你会看到项目设置界面**
   - 顶部有几个标签：`General`、`Signing & Capabilities`、`Resource Tags` 等
   - 如果没有看到这些标签，说明你点击的位置不对，重新点击项目名称（蓝色图标）

2. **在左侧的 TARGETS 列表中，找到并点击 `WorkSurvivalGuide`**
   - TARGETS 列表在中间面板的左侧
   - 应该只有一个 target，名称是 `WorkSurvivalGuide`
   - 点击后，中间面板会显示该 target 的设置

### 第三步：找到 Deployment Info

1. **点击顶部的 `General` 标签**
   - 如果 `General` 标签已经是选中状态（高亮显示），可以跳过这一步

2. **向下滚动，找到 `Deployment Info` 部分**
   - 这个部分通常在 `General` 标签页的中间位置
   - 你会看到类似这样的内容：
     ```
     Deployment Info
     iOS: 17.0 (或 17.1, 17.2 等)
     ```

### 第四步：修改 iOS 版本

1. **找到 `iOS` 这一行**
   - 在 `Deployment Info` 部分，第一行就是 `iOS`
   - 右侧会显示当前的版本号（可能是 17.0 或其他版本）

2. **点击版本号旁边的下拉菜单（向下的小箭头）**
   - 点击后会弹出一个版本列表

3. **选择 `16.0`**
   - 在列表中向下滚动，找到 `16.0`
   - 点击选择它

4. **确认修改**
   - 选择后，版本号会自动更新为 `16.0`
   - 不需要点击保存，Xcode 会自动保存

### 第五步：验证设置

1. **检查是否设置成功**
   - 在 `Deployment Info` 部分，`iOS` 应该显示为 `16.0`
   - 如果显示正确，说明设置成功！

## ⚠️ 常见问题

### 问题 1: 找不到项目名称（蓝色图标）

**解决方法**：
- 确保左侧面板（项目导航器）是打开的
- 如果看不到，按 `Cmd + 1` 打开
- 项目名称应该在最顶部，显示为蓝色文件夹图标

### 问题 2: 点击后看不到 General 标签

**解决方法**：
- 确保你点击的是项目名称（蓝色图标），不是文件夹
- 在中间面板的顶部，应该能看到多个标签
- 如果还是看不到，尝试点击项目名称旁边的 `>` 展开，然后再点击项目名称

### 问题 3: 找不到 Deployment Info

**解决方法**：
- 确保你点击了 `General` 标签
- 在 `General` 标签页中，向下滚动查找
- `Deployment Info` 通常在中间位置，不在最顶部

### 问题 4: 下拉菜单中没有 16.0

**解决方法**：
- 确保你的 Xcode 版本支持 iOS 16.0
- 如果确实没有，可以选择最接近的版本（如 16.1、16.2）
- 但建议使用 iOS 16.0 或更高版本
