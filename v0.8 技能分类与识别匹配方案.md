# v0.8 技能分类与识别匹配方案

## 一、技能分类体系

### 顶级分类（3 大类）

| 分类 | 英文标识 | 说明 |
|------|---------|------|
| 职场 | workplace | 工作相关场景，含 5 个分析维度 |
| 家庭 | family | 家庭关系与教育场景 |
| 个人 | personal/emotion | 情绪识别与心理健康监控 |

### 职场（workplace）— 5 个维度，19 个子技能

#### 维度 1：角色方位（role_position）
从权力关系与角色定位视角分析对话。

| 子技能 | 英文 ID | 关键词 | 分析焦点 |
|--------|---------|--------|----------|
| 向上管理 | managing_up | 老板、领导、上级、汇报、请示 | 权力不对等下的上行沟通策略 |
| 向下管理 | managing_down | 下属、团队、安排、委派、指导 | 领导力、授权与反馈技巧 |
| 横向协作 | peer_collaboration | 同事、协作、配合、跨部门 | 无权威影响力、利益交换 |
| 对外沟通 | external_communication | 客户、合作方、外部、销售 | 商务边界、客户关系维护 |

**技能文件**: `skills/workplace_role/SKILL.md`

## 二、识别匹配流程

### 整体流程

```
录音上传 → 转录 → Router 场景分类（1次LLM调用）
                    ├── 顶级分类: workplace / family / other
                    └── 职场维度识别: 2-4个维度+子技能
                          ↓
                    技能匹配（match_skills）
                    ├── workplace: 按维度匹配对应 SKILL
                    ├── family: 按 category 匹配
                    └── emotion: 特殊规则（始终/关键词触发）
                          ↓
                    并行执行匹配的技能
                    ├── 每个维度技能注入 matched_sub_skill
                    ├── 独立 LLM 调用生成分析
                    └── 输出 visual + strategies
                          ↓
                    构建 skill_cards（每技能一张卡片）
                    ├── category / dimension / matched_sub_skill
                    └── content（visual + strategies / emotion / mental_health）
                          ↓
                    iOS 多维度展示
                    ├── 场景 Tab（职场/家庭/个人）
                    ├── 策略图轮播（合并同场景所有 visual）
                    └── 手风琴面板（维度名 · 子技能名）
```

### Router 分类输出格式

```json
{
  "scenes": [
    {"category": "workplace", "confidence": 0.95, "reasoning": "对话涉及项目汇报"},
    {"category": "family", "confidence": 0.3, "reasoning": "提到了家庭安排"}
  ],
  "primary_scene": "workplace",
  "workplace_dimensions": [
    {"dimension": "role_position", "sub_skill": "managing_up", "sub_skill_name": "向上管理", "confidence": 0.9, "reasoning": "对话是向领导汇报工作"},
    {"dimension": "scenario", "sub_skill": "presentation", "sub_skill_name": "汇报展示", "confidence": 0.85, "reasoning": "正在做工作汇报"},
    {"dimension": "psychology", "sub_skill": "defensive", "sub_skill_name": "防御型", "confidence": 0.7, "reasoning": "用户处于防御姿态"}
  ]
}
```

### 匹配规则

1. **职场场景**: 当 `workplace` 命中时，读取 `workplace_dimensions` 列表，按 `dimension` 字段匹配对应的维度技能（`workplace_role`、`workplace_scenario` 等），通常命中 2-4 个维度
2. **家庭场景**: 当 `family` 命中时，匹配所有 `category=family` 的技能
3. **情绪技能**: `emotion_recognition` 始终运行；`depression_prevention` 仅在用户话术命中关键词时触发
4. **关键词兜底**: 即使 LLM 未识别，对话中出现职场/家庭角色关键词时自动补充对应场景
5. **置信度阈值**: 场景置信度 > 0.3 才参与匹配，最多取 3 个场景

## 四、iOS 客户端 UI 结构

### 数据模型扩展

- `SkillCard` 新增字段: `category`、`dimension`、`matchedSubSkill`
- `StrategyAnalysisResponse` 新增字段: `matchedScenes`
- 新增枚举: `SceneCategory`（职场/家庭/个人）、`WorkplaceDimension`（5个维度）
