# iOS 创建模拟器 OS 版本选择问题

## 🎯 当前情况

从截图看：
- ✅ iOS 18.5 + iOS 18.6 Simulator 已经安装（显示 "8.84 GB on disk"）
- ⏳ iOS 17.0 Simulator 正在下载中（12%）
- ❌ 创建模拟器时，OS Version 下拉菜单显示 "✓ No Runtimes"

---

## ✅ 解决方案

### 方法 1: 关闭对话框，重新打开（推荐）

1. **点击 "Cancel" 按钮**
   - 关闭创建模拟器对话框

2. **等待几秒钟**
   - 让系统刷新

3. **重新点击 `+` 按钮**
   - 打开创建模拟器对话框

4. **再次点击 "OS Version" 下拉菜单**
   - 现在应该能看到 iOS 18.6 选项了

5. **选择 iOS 18.6**
   - 点击选择

6. **点击 "Create" 按钮**
   - 创建模拟器

---

### 方法 2: 如果还是没有选项，检查已安装的 Runtime

1. **点击 "Cancel" 关闭对话框**

2. **在终端运行命令检查**：
   ```bash
   xcrun simctl list runtimes
   ```

3. **查看输出**
   - 应该能看到已安装的 iOS Runtime
   - 查找 iOS 18.6

4. **如果能看到 iOS 18.6，说明已安装**
   - 重新打开创建模拟器对话框
   - 应该能看到选项了

---

### 方法 3: 使用已存在的模拟器（如果列表中有）

1. **查看左侧模拟器列表**
   - 在 Devices and Simulators 窗口左侧
   - 查找是否有 iOS 18 的模拟器

2. **如果有，直接使用**
   - 点击选择
   - 在 Xcode 顶部工具栏选择它
   - 运行项目

---

## 🔍 详细操作步骤

### 步骤 1: 关闭当前对话框

1. **在创建模拟器对话框中**
   - 点击 "Cancel" 按钮
   - 关闭对话框

### 步骤 2: 等待 iOS 17 下载完成（可选）

如果你想要 iOS 17 模拟器：
- 等待 iOS 17.0 Simulator 下载完成（当前 12%）
- 下载完成后，重新打开创建对话框
- 应该能看到 iOS 17.0 选项

### 步骤 3: 重新创建模拟器

1. **点击 `+` 按钮**
   - 重新打开创建模拟器对话框

2. **配置**：
   - **Simulator Name**: `iPhone 14`（或自定义名称）
   - **Device Type**: `iPhone 14`（保持不变）
   - **OS Version**: 点击下拉菜单

3. **查看 OS Version 选项**
   - 现在应该能看到：
     - `iOS 18.6`（已安装）
     - `iOS 17.0`（如果下载完成）

4. **选择 iOS 18.6**
   - 点击选择

5. **点击 "Create" 按钮**
   - 创建模拟器

---

## 🆘 如果还是没有选项

### 问题排查

1. **检查 Xcode 版本**
   - 菜单栏：`Xcode` → `About Xcode`
   - 查看版本号

2. **重启 Xcode**
   - 完全退出 Xcode
   - 重新打开
   - 再次尝试

3. **使用命令行创建模拟器**

如果图形界面有问题，可以用命令行：

```bash
# 查看可用的设备类型
xcrun simctl list devicetypes

# 查看已安装的 Runtime
xcrun simctl list runtimes

# 创建模拟器（替换为实际的设备类型和 Runtime）
xcrun simctl create "iPhone 14 (iOS 18)" "iPhone 14" "iOS 18.6"
```
