# 设置 Deployment Target - 最终方法（100% 有效）

## 🎯 最直接的方法：通过项目导航器

### 详细步骤：

#### 第 1 步：找到项目导航器

1. **查看 Xcode 窗口的最左侧**
   - 应该有一个垂直的面板，显示文件列表
   - 这就是**项目导航器**

2. **如果看不到左侧面板**：
   - 按快捷键：`Command + 1`（Mac 上按 `Cmd + 1`）
   - 或者点击顶部菜单栏的 `View` → `Navigators` → `Show Project Navigator`

#### 第 2 步：找到项目名称并展开

1. **在项目导航器的最顶部，找到项目名称**
   - 显示为蓝色文件夹图标 📁
   - 名称应该是 `WorkSurvivalGuide`

2. **点击项目名称旁边的** `>` **（小箭头）**
   - 这会展开项目结构
   - 展开后，你应该能看到类似这样的结构：
   ```
   📁 WorkSurvivalGuide
      📁 WorkSurvivalGuide
      📁 Models.swift
      TARGETS
         WorkSurvivalGuide  ← 这个很重要！
      PROJECT
         WorkSurvivalGuide
   ```

#### 第 3 步：点击 TARGETS 下的 WorkSurvivalGuide

1. **在 `TARGETS` 部分，找到 `WorkSurvivalGuide`**
   - 应该只有一个 target，名称是 `WorkSurvivalGuide`

2. **单击 `WorkSurvivalGuide`**
   - 点击后，**中间的主编辑区域**会发生变化
   - 应该显示项目设置界面，而不是代码

#### 第 4 步：确认界面变化

点击 target 后，中间面板应该显示：

```
┌─────────────────────────────────────────┐
│  General | Signing & Capabilities | ... │ ← 这些标签
│                                          │
│  [项目信息]                              │
│                                          │
│  Deployment Info                        │
│  iOS: 17.0 ▼                            │ ← 这里！
│                                          │
└─────────────────────────────────────────┘
```

**如果你看到这些标签**（General、Signing & Capabilities 等），说明操作正确！

**如果还是看到代码**，说明点击的位置不对，请重新点击 `TARGETS` 下的 `WorkSurvivalGuide`。

#### 第 5 步：点击 General 标签

1. **在中间面板的顶部，点击 `General` 标签**
   - `General` 标签应该在第一个位置
   - 点击后，标签会高亮显示

#### 第 6 步：找到 Deployment Info

1. **在 `General` 标签页中，向下滚动**
   - `Deployment Info` 部分通常在页面的**中间位置**

2. **查找 `Deployment Info` 标题**
   - 你应该能看到：
   ```
   Deployment Info
   iOS: 17.0 ▼  ← 这里显示当前版本
   ```

#### 第 7 步：修改 iOS 版本

1. **点击 `iOS` 右侧的版本号**（或下拉箭头 ▼）
2. **在下拉列表中选择 `16.0`**
3. **完成！** 版本会自动更新为 `16.0`

## 🎯 如果还是找不到，试试这个方法

### 方法 2：通过搜索功能

1. **按快捷键打开搜索**：
   - 在 Mac 上按：`Command + Shift + O`（`Cmd + Shift + O`）
   - 这会打开快速打开对话框

2. **输入搜索关键词**：
   - 输入：`deployment`
   - 或者输入：`target`

3. **选择相关选项**：
   - 在搜索结果中，找到与 `Deployment Target` 相关的选项
   - 点击它，会直接跳转到设置位置

---

## 🎯 方法 3：检查项目结构

如果项目导航器中没有显示 `TARGETS`，可能是项目结构有问题：

### 步骤：

1. **确保项目已正确打开**
   - 在 Finder 中，找到项目文件夹
   - 双击 `WorkSurvivalGuide.xcodeproj` 文件
   - 这会在 Xcode 中打开项目

2. **检查项目导航器**
   - 项目名称应该在导航器的最顶部
   - 点击项目名称旁边的 `>` 应该能展开

---

## 🎯 方法 4：通过 Build Settings（高级方法）

如果以上方法都不行，可以尝试通过 Build Settings：

### 步骤：

1. **在项目导航器中，点击项目名称（蓝色图标）**

2. **在中间面板，点击 `Build Settings` 标签**
   - 如果看不到这个标签，可能需要先点击项目名称

3. **在搜索框中输入**：`deployment target`
   - 在 Build Settings 页面的顶部，有一个搜索框

4. **找到 `iOS Deployment Target`**
   - 在搜索结果中，找到 `iOS Deployment Target`
   - 双击版本号，修改为 `16.0`

## 📸 你现在的界面应该是什么样的？

**设置 Deployment Target 时，你的界面应该是**：

```
┌─────────────────────────────────────────────────────┐
│  [左侧导航器]  [中间：项目设置]  [右侧：可能没有]    │
│  ┌──────────┐  ┌──────────────────────────────┐   │
│  │ 📁 项目  │  │ General | Signing | ...     │   │
│  │  名称    │  │                              │   │

│  │    >     │  │  iOS: [16.0 ▼]               │   │
│  │ TARGETS  │  │                              │   │
│  │  Work... │  │                              │   │
│  └──────────┘  └──────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

**关键点**：
- 左侧：项目导航器，显示 `TARGETS` → `WorkSurvivalGuide`
- 中间：项目设置界面，显示 `General` 标签和 `Deployment Info`
