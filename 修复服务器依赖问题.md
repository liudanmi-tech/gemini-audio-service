# 修复服务器依赖问题

## 问题：缺少 Python 依赖包

## 解决方案

### 方法 1: 使用虚拟环境（推荐）

```bash
cd ~/gemini-audio-service

# 检查是否有虚拟环境
ls -la | grep venv

# 如果有 venv 目录，激活它
source venv/bin/activate

# 如果没有 venv，创建一个
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip3 install -r requirements.txt

# 启动服务
nohup python3 main.py > ~/gemini-audio-service.log 2>&1 &
sleep 3

# 检查服务是否启动
ps aux | grep 'python.*main.py' | grep -v grep
```

### 方法 2: 直接安装依赖（如果没有虚拟环境）

```bash
cd ~/gemini-audio-service
pip3 install -r requirements.txt

# 启动服务
nohup python3 main.py > ~/gemini-audio-service.log 2>&1 &
sleep 3

# 检查服务是否启动
ps aux | grep 'python.*main.py' | grep -v grep
```

### 方法 3: 使用 pip3 安装（如果 requirements.txt 有问题）

```bash
cd ~/gemini-audio-service
pip3 install fastapi uvicorn google-generativeai python-dotenv pydantic
```
