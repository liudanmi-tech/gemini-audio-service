## 录音匹配技能流程

```
录音上传
  │
  ▼
① classify_scene（LLM 调用）
  │  识别命中的一级场景（workplace/family/personal）
  │  若命中 workplace，进一步识别 workplace_dimensions：
  │  返回 dimension + sub_skill + sub_skill_name
  │  例：{dimension: "role_position", sub_skill: "managing_up", sub_skill_name: "向上管理"}
  │
  ▼
② match_skills（技能匹配）
  │  根据场景 + 维度，从 skills 表中找到对应的维度技能
  │  将 sub_skill_name 写入 matched_sub_skill 字段
  │
  ▼
③ execute_skill（技能执行）
  │  每个命中的技能独立调用 LLM 生成分析
  │  matched_sub_skill 注入 prompt 上下文
  │
  ▼
④ 构建 skill_cards 写入数据库
  │  skill_id: "workplace_role"
  │  skill_name: "角色方位分析"
  │  matched_sub_skill: "向上管理"  ← 与技能库名称一致
  │  dimension: "role_position"
  │  category: "workplace"
  │
  ▼
⑤ iOS 展示
   accordionTitle 优先显示 matched_sub_skill（"向上管理"）
   与技能库中的卡片名称保持一致
```

## 关键字段对照

| 数据位置 | 字段 | 示例值 | 说明 |
|---------|------|--------|------|
| skills 表 | skill_id | `workplace_role` | 维度技能的唯一标识 |
| skills 表 | name | `角色方位分析` | 维度技能的中文名称 |
| skills 表 metadata | sub_skills[].id | `managing_up` | 子技能ID |
| skills 表 metadata | sub_skills[].name | `向上管理` | 子技能中文名称（技能库展示用） |
| strategy_analysis | skill_cards[].skill_id | `workplace_role` | 分析结果中命中的维度技能 |
| strategy_analysis | skill_cards[].skill_name | `角色方位分析` | 维度技能名称 |

| strategy_analysis | skill_cards[].dimension | `role_position` | 维度标识 |
| strategy_analysis | skill_cards[].category | `workplace` | 一级场景分类 |
